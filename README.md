# input-cycle-bot

LINE Bot で今日のコンディションを伝えるだけで、LLM に貼るための「学習タスクプロンプト」を自動生成してくれる Bot です。

音声でもテキストでも OK。「疲れた」「元気」みたいにひとこと送るだけで、コンディションに合わせたサイクル数・タスク内容を提案するプロンプトが返ってきます。

生成されたプロンプトを Claude / ChatGPT / Gemini など、お好みの LLM にそのまま貼り付けて使います。

## 概要

### このプロジェクトが解決する課題

#### 毎日の学習計画を立てるのが面倒

「今日は何を勉強しよう」と考えるところから始めると、それだけで時間と気力を消耗してしまいます。特に受験勉強のように複数の教科をバランスよく回す必要がある場合、計画を立てること自体がハードルになりがちです。

このBotは、コンディションを一言伝えるだけで、LLM がタスクを提案してくれるプロンプトを自動で組み立てます。

#### コンディションによってやれることが違う

元気な日と疲れた日では、こなせるタスクの量も質も変わります。「今日は頭痛がする」という日に60分の問題演習は現実的ではありません。

コンディションレベル（1-5）に応じてサイクル数が自動調整されるので、無理のない学習計画が生成されます。

#### LLM に毎回コンテキストを伝えるのが大変

Claude や ChatGPT に学習相談をするとき、「最近何をやったか」「ブログの進捗はどうか」などのコンテキストを毎回手入力するのは非効率です。

このBotは直近7日間の学習履歴やブログパイプラインの状態を自動でプロンプトに組み込むので、LLM に貼るだけで文脈を踏まえた提案が得られます。

### アーキテクチャ

#### 技術スタック

| 要素 | 技術 |
|---|---|
| ランタイム | Cloudflare Workers |
| フレームワーク | Hono |
| 言語 | TypeScript（strict mode） |
| DB | Cloudflare D1（SQLite） |
| 音声文字起こし | Cloudflare Workers AI（`@cf/openai/whisper`） |
| メッセージング | LINE Messaging API |

#### 処理フロー

```
ユーザー（LINE）
  ↓ テキスト or 音声
LINE Webhook（POST /webhook/line）
  ↓ 署名検証（HMAC-SHA256）
  ↓ 音声の場合 → Workers AI (Whisper) で文字起こし
コマンドルーター
  ├── 「記録 ...」→ 学習記録を DB に保存
  ├── 「ブログ...」→ ブログパイプライン管理
  ├── 「履歴」→ 直近7日間の記録を表示
  ├── 「ヘルプ」→ 使い方を表示
  └── その他 → コンディション判定 → プロンプト生成
  ↓
LINE Reply API でプロンプトを返信
  ↓
ユーザーがプロンプトを Claude / ChatGPT / Gemini に貼り付け
```

#### ディレクトリ構成

```
src/
├── index.ts                    # エントリポイント（Hono アプリ）
├── routes/
│   └── webhook.ts              # LINE Webhook エンドポイント
├── services/
│   ├── line.ts                 # LINE API（署名検証・音声取得・返信）
│   ├── transcribe.ts           # Workers AI による音声文字起こし
│   ├── condition.ts            # ルールベースのコンディション判定
│   └── prompt-builder.ts       # LLM 用プロンプト生成
├── commands/
│   ├── router.ts               # メッセージの振り分け
│   ├── record.ts               # 「記録」コマンド
│   ├── blog.ts                 # 「ブログ」コマンド
│   └── history.ts              # 「履歴」コマンド
├── db/
│   ├── schema.sql              # テーブル定義
│   └── queries.ts              # DB クエリ関数
├── utils/
│   └── date.ts                 # JST 日付ヘルパー
└── types/
    └── index.ts                # 型定義
```

### 主な機能

#### コンディション判定（ルールベース）

##### キーワードスコアリング

テキストに含まれるキーワードをスコアリングして、ベーススコア 3 から加減算します。LLM は一切使いません。

| 種別 | キーワード例 | スコア |
|---|---|---|
| 強いネガティブ | ぐったり、無理、限界、頭痛 | -2 |
| ネガティブ | 疲れ、しんどい、だるい、眠い | -1 |
| ポジティブ | 元気、いける、大丈夫、普通 | +1 |
| 強いポジティブ | めっちゃ元気、やる気、絶好調、最高 | +2 |

##### 時間帯補正

JST 22時以降は自動で -1 の補正がかかります。夜遅い時間は無理しないように。

##### 5段階レベルとサイクル数の対応

| レベル | サイクル数 | 目安時間 |
|---|---|---|
| 1 | 1サイクル | 10-15分 |
| 2 | 2サイクル | 20-30分 |
| 3 | 3サイクル | 30-45分 |
| 4 | 4サイクル | 40-60分 |
| 5 | 5-6サイクル | 60-90分 |

#### プロンプト自動生成

##### サイクル単位のタスク提案

1サイクル = 10〜20分で完結する学習タスク。途中で終わっても成果が残るように設計されています。

##### 学習履歴の自動反映

直近7日間の学習記録がプロンプトに自動で含まれるため、LLM が「最近やっていない教科」を判断できます。

##### ブログパイプラインの組み込み

現在進行中のブログテーマとそのステータスがプロンプトに含まれるため、次にやるべきアクションが提案されます。

#### 音声入力対応

##### Cloudflare Workers AI（Whisper）による文字起こし

LINE の音声メッセージを Cloudflare Workers AI の Whisper モデルで文字起こしします。無料枠内で利用可能です。

#### 学習記録の管理

##### 日次ログの保存（UPSERT）

同じ日に複数回コンディションを送信しても、最新の入力で上書きされます。

##### サマリーの記録

「記録」コマンドで学習内容を保存。翌日以降のプロンプト生成に活用されます。

#### ブログパイプライン管理

##### ステータス管理（backlog → researching → testing → writing → published）

5段階のステータスでブログ執筆の進捗を管理できます。

##### テーマの追加・更新・一覧表示

LINE のチャット上からブログテーマの CRUD 操作が可能です。

### Bot が LLM API を使わない理由

このBotは意図的に LLM API を呼び出しません。理由は以下の通りです。

- **コスト**: API 呼び出しの従量課金が発生しない
- **速度**: LLM の生成待ち（数秒〜十数秒）がなく、即座にレスポンスが返る
- **自由度**: 生成されたプロンプトをユーザーが好きな LLM（Claude / ChatGPT / Gemini）に貼れる
- **サブスク活用**: 多くのユーザーが既に LLM のサブスクを持っているため、Bot 側で API を叩く必要がない

唯一の AI 使用箇所は、音声メッセージの文字起こし（Cloudflare Workers AI / Whisper）のみです。これは無料枠で収まります。

## セットアップ手順

### 前提条件

#### 必要なアカウント

##### Cloudflare アカウント

[Cloudflare](https://www.cloudflare.com/) の無料プランで OK です。Workers、D1、Workers AI すべて無料枠内で動作します。

##### LINE Developers アカウント

[LINE Developers](https://developers.line.biz/) でアカウントを作成し、Messaging API チャネルを用意します。

#### 必要なツール

##### Node.js（v18 以上）

```bash
node -v
# v18.x.x 以上であること
```

##### Wrangler CLI

```bash
npm install -g wrangler
wrangler login
```

### LINE チャネルの作成

#### LINE Developers コンソールでの操作

##### プロバイダーの作成

LINE Developers コンソールにログインし、新しいプロバイダーを作成します。

##### Messaging API チャネルの作成

作成したプロバイダーの中で「Messaging API」チャネルを作成します。

##### チャネルアクセストークンの発行

「Messaging API 設定」タブからチャネルアクセストークン（長期）を発行します。後のシークレット設定で使います。

##### チャネルシークレットの確認

「チャネル基本設定」タブからチャネルシークレットを確認します。こちらも後で使います。

### ワンコマンドセットアップ（推奨）

#### setup スクリプトの実行

```bash
npm run setup
```

#### スクリプトが行うこと

##### npm install による依存パッケージのインストール

Hono、TypeScript、Vitest などの依存パッケージがインストールされます。

##### Cloudflare ログイン確認

`wrangler whoami` でログイン状態を確認します。未ログインの場合はエラーで停止します。

##### D1 データベースの作成と wrangler.toml への反映

`wrangler d1 create input-cycle-bot-db` を実行し、返却された `database_id` を `wrangler.toml` に自動で書き込みます。

##### スキーマの適用（テーブル作成）

`src/db/schema.sql` を使って `daily_logs` テーブルと `blog_pipeline` テーブルを作成します。

##### LINE シークレットの設定（対話式）

`LINE_CHANNEL_ACCESS_TOKEN` と `LINE_CHANNEL_SECRET` を対話形式で入力します。

##### Cloudflare Workers へのデプロイ

`wrangler deploy` でワーカーをデプロイします。

##### 初期データの投入

ブログパイプラインに初期テーマが登録されます。

## ユースケース

### 日常の学習サイクル

#### 朝のコンディション入力から学習開始まで

##### テキストで「まあまあ元気」と送信する場合

「まあまあ」と「元気」がポジティブキーワードにマッチし、レベル 5（ベース3 + 1 + 1）と判定されます。5-6サイクル（60-90分）の学習プロンプトが生成されます。

##### 音声メッセージで「今日はちょっと疲れた」と送る場合

音声が Whisper で文字起こしされ、「疲れ」がネガティブキーワードにマッチ。レベル 2（ベース3 - 1）となり、2サイクル（20-30分）の軽めのプロンプトが生成されます。

##### 生成されたプロンプトを LLM に貼り付ける

返信されたプロンプトをそのまま Claude / ChatGPT / Gemini に貼り付けると、サイクル単位の具体的なタスク提案が得られます。

#### 学習後の記録

##### 「記録 英語 長文読解15分」のようにテキストで送信

```
記録 英語 長文読解15分
```

→ `✅ 記録しました（2026-03-04 水）` と確認メッセージが返ります。

##### 複数の学習内容をまとめて記録

```
記録 数学 二次関数15分、英語 単語10分
```

1回のメッセージで複数教科をまとめて記録できます。

#### 一日の振り返り（履歴確認）

##### 「履歴」で直近7日間の学習記録を表示

```
履歴
```

→ 日付ごとにコンディションレベルと学習サマリーが一覧表示されます。

### ブログ執筆の進捗管理

#### 新しいテーマの追加

##### 「ブログ追加 CIBA フロー決済承認」のように送信

```
ブログ追加 CIBA フロー決済承認
```

→ `✅ ブログテーマを追加しました [ID:1] CIBA フロー決済承認 ステータス: backlog`

#### ステータスの更新

##### 調査フェーズへの移行（researching）

```
ブログ更新 1 researching
```

##### 検証フェーズへの移行（testing）

```
ブログ更新 1 testing 実機でCIBAフロー検証済み
```

##### 執筆フェーズへの移行（writing）

```
ブログ更新 1 writing
```

##### 公開済みへの変更（published）

```
ブログ更新 1 published
```

##### メモの追加

ステータス更新時に3番目の引数としてメモを追加できます。調査メモや気づきの記録に便利です。

#### パイプライン全体の確認

##### 「ブログ一覧」でステータス順に全テーマを表示

```
ブログ一覧
```

→ 執筆中 → 検証中 → 調査中 → バックログ → 公開済みの順にソートされて表示されます。

### コンディション別の活用パターン

#### コンディション 1-2（低調時）

##### 受動的タスク優先（読む・見る・復習）

LLM が提案するタスクは、教科書を読む・動画を見る・既習内容の復習など、負荷の軽いものが中心になります。

##### サイクル数を抑えて短時間で終了

1-2サイクル（10-30分）で終わるので、体調が悪い日でも「何もやらなかった」を避けられます。

#### コンディション 3（普通）

##### バランス型のタスク配分

受験勉強とブログの両方を含む、バランスの取れたタスク提案が生成されます。

#### コンディション 4-5（好調時）

##### 能動的タスク優先（問題を解く・検証する・書く）

問題演習、コード実装、ブログ執筆など、アウトプット中心のタスクが提案されます。

##### 多めのサイクル数で集中学習

4-6サイクル（40-90分）のタスクが提案され、好調な日を最大限活用できます。

### 夜遅い時間の入力

#### 22時以降の時間帯補正による自動調整

##### 「元気」と送っても通常より控えめなサイクル数に

22時以降は自動で -1 の補正がかかるため、「元気」と送ってもレベル 3（通常は 4）に調整されます。夜更かし学習の防止に。

### 同日の再入力（やり直し）

#### UPSERT による上書き更新

##### 朝と夜でコンディションが変わった場合の再送信

同じ日にコンディションを送り直すと、DB 上のレコードが上書きされます。朝は元気だったけど午後に疲れた、という場合に再入力できます。

## 使った感想

### 良かった点

#### LLM API を使わないことの利点

##### ランニングコストがほぼゼロ

Cloudflare Workers、D1、Workers AI のすべてが無料枠内で運用できます。個人的にはこれが一番大きなメリットだと感じています。API 課金を気にせず、毎日気軽に使えます。

##### レスポンスが速い（LLM の生成待ちがない）

プロンプト生成はルールベースの文字列組み立てなので、LLM の推論を待つ必要がありません。LINE にメッセージを送ってから返信が来るまで、体感で1秒もかかりません。

##### プロンプトを自分の好きな LLM に使える自由度

生成されたプロンプトは汎用的なので、Claude でも ChatGPT でも Gemini でも使えます。「今日は Claude の方が調子良さそう」みたいな使い分けができるのが意外と便利です。

#### Cloudflare Workers + D1 の開発体験

##### Wrangler CLI によるスムーズな開発フロー

`wrangler dev` でローカル開発 → `wrangler deploy` で本番デプロイ、という流れがシンプルで快適です。

##### D1 の SQL ベースのシンプルさ

SQLite ベースなので、複雑な ORM を使わずに生 SQL で書けるのが個人開発には合っています。UPSERT も `ON CONFLICT DO UPDATE` で直感的に書けます。

##### Workers AI（Whisper）の無料枠で音声対応できる手軽さ

音声文字起こしのために外部 API を契約する必要がなく、Cloudflare のバインディング1つで完結するのは大きな魅力です。

#### サイクル単位の学習設計

##### 10-20分の小さなタスクに分解される安心感

「1時間勉強しよう」だと心理的ハードルが高いですが、「10分だけ」なら始められる。この仕組みが継続のカギだと思います。

##### コンディションに応じた自動調整で無理しない

体調が悪い日に無理な計画を立てて挫折する、というパターンを防げます。「今日は1サイクルだけ」でも、ゼロよりは確実に前に進んでいます。

#### LINE という入力チャネルの便利さ

##### アプリを開く必要がない（日常の延長で使える）

専用アプリのインストールが不要で、普段使っている LINE からそのまま使えるのが楽です。

##### 音声入力で手が離せない時でも使える

通勤中や食事中でも、音声メッセージで「今日は元気」と送るだけでプロンプトが生成されます。

### 課題・改善したい点

#### LINE 無料プランの月200通制限

##### 1日6-7通が上限の目安

コンディション入力 + 記録 + 履歴確認で1日3-4通は使うので、残りの余裕があまりありません。

##### 記録・ブログコマンドも1通としてカウントされる

Bot の返信はすべて1通としてカウントされるため、ブログ管理を頻繁に使うと通数が圧迫されます。

#### キーワードベースのコンディション判定の限界

##### ニュアンスの読み取りが難しいケース

「まあ、元気じゃないこともないけど...」のような微妙な表現は、キーワードマッチだけでは正確に判定しにくい場合があります。

##### キーワード辞書の継続的なチューニングが必要

自分の語彙に合わせてキーワードを追加・調整していく必要があります。最初のうちは「あ、このキーワード登録されてないな」と感じる場面があるかもしれません。

#### 学習軸がハードコードされている点

##### 現状は「受験勉強」「認証認可ブログ」の2軸固定

プロンプトテンプレートに学習テーマが直接書かれているため、他のテーマで使いたい場合はコードを変更する必要があります。

##### 他の学習テーマに対応するにはコード変更が必要

`src/services/prompt-builder.ts` を編集してテーマを書き換えれば対応可能ですが、DB から動的に読む仕組みがあるとより汎用的になります。

### 技術的に学びになった点

#### Hono フレームワークの軽量さと型安全性

Hono は Express に似た書き心地でありながら、Cloudflare Workers にネイティブ対応している点が良かったです。`Bindings` 型を `Hono<{ Bindings: Bindings }>` で渡すだけで環境変数の型が効くのは便利。

#### LINE Messaging API の署名検証の仕組み（HMAC-SHA256）

Web Crypto API を使った HMAC-SHA256 の署名検証を実装することで、Webhook のセキュリティの仕組みを深く理解できました。

#### D1 の UPSERT パターン（ON CONFLICT DO UPDATE）

SQLite の `ON CONFLICT` 句を使った UPSERT パターンは、同日の再入力を上書きするという要件にぴったりでした。

#### Workers AI のバインディングによる AI 呼び出し

`c.env.AI.run()` のように、バインディング経由で AI モデルを呼び出せるのは Cloudflare ならではの体験です。API キーの管理が不要なのも嬉しいポイント。

### 今後やりたいこと

#### 学習軸の動的設定（DB管理化）

現在ハードコードされている学習テーマを DB で管理し、LINE のコマンドから追加・変更できるようにしたいです。

#### 週次レポートの自動生成

1週間分の学習記録をまとめて、振り返り用のサマリーを自動生成する機能を追加したいです。

#### コンディション判定の精度向上

##### カスタムキーワードの追加機能

ユーザーが自分の語彙に合わせてキーワードを追加できる機能があると、判定精度が上がりそうです。

#### Flex Message 対応（リッチな LINE UI）

現在はテキストのみの返信ですが、LINE の Flex Message を使えば、サイクルごとにカード形式で表示するなど、より見やすい UI にできそうです。

## 制約事項

- LINE Messaging API の無料プランは**月200通まで**（1日6-7通が目安）
- LINE テキストメッセージは**5000文字制限**（超過時は自動分割送信）
- D1 の `datetime('now')` は **UTC**（日付管理は JST 変換済みの TEXT カラムで実施）
- LINE Webhook は**必ず 200 を返す**必要あり（エラー時も 200 を返さないと再送が繰り返される）
